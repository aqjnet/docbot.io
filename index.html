01 · The gap

Aconex notifies.
Nobody chases.

Client-mandated document platforms tell you a document arrived. They don't notice what didn't arrive — the transmittal that's nine days overdue, the consultant who went quiet, the reply sitting in an inbox that closes the loop. That work lands on people.

The cost

AUD 8–11k/mo

Loaded cost of a document controller — much of it spent scanning registers and writing the same chase email again.

The blind spot

Overdue is silent

Platforms fire events when documents move. The expensive failures are the ones where nothing moved — and no event ever fires.

The loop

Chase → reply → update

The reply comes back as free text. Someone has to read it, map it to the register, and make the entry. That's the loop DocBot closes.

02 · How it works

A deterministic spine.
A model used like a scalpel.

Most of the loop is plain code — reproducible run-to-run, defensible line-by-line. The model fires at exactly four decision points where the register is genuinely ambiguous. Anything consequential waits at a single human checkpoint.

Four model nodes, no more

extract.interpret reads reply prose and maps it to a register change — the irreducible one. classify.inbound, infer.owner and exception.check handle triage, blank metadata, and knowing when not to fire.

The spine is boring on purpose

Overdue maths, cadence rules, lookups, the send, the commit, the audit write — plain code. The graph's edges are deterministic; the model only ever lives inside a node.

One gate, everything logged

Low-risk internal nudges auto-send. Anything client-facing or contractual parks at the approval gate until a human signs off. Approve or reject, the trail records it.

04 · The stance

Built to be trusted
with a paper trail.

Document control is an audit-sensitive job. The architecture treats that as a feature, not a compliance afterthought.

/01

Overlay, not system of record

Aconex and Procore own the register. DocBot reads it and acts on it — chases, interprets, proposes. It never tries to replace the CDE your client mandated.

/02

Deterministic spine, LLM scalpel

If a step can be plain code, it is plain code. The model is reserved for the four judgment calls where prose meets the register — and each call is logged with its confidence.

/03

Propose, not commit

The agent chases autonomously — that's low-risk. But a register entry or a client-facing message is contractual weight, so it parks at the gate until a human approves it.

/04

Audit-grade evidence trail

Every scan, every model judgment, every approval and every send writes an immutable audit row. When someone asks "why did this change?", the answer is one query away.
